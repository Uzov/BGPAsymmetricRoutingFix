# BGP Asymmetric Routing Fix (FortiGate + SNMP + API)

## 📌 Описание

Скрипт предназначен для устранения проблемы **асимметричной маршрутизации** при работе с двумя интернет-провайдерами.

### Провайдеры

- Основной канал: **Avantel**
- Резервный / альтернативный: **Beeline**

Оба провайдера анонсируют маршрут по умолчанию (`0.0.0.0/0`).

---

## ❗ Проблема

В текущей схеме возникает асимметрия:

- Исходящий трафик уходит через **Avantel**
- Обратный трафик от клиентов **Beeline** приходит через **Beeline**

### Последствия:

- асимметричная маршрутизация
- разрыв TCP-сессий на firewall
- недоступность сервисов для клиентов Beeline

---

## ❗ Почему не помогает BGP prepend

Увеличение AS-path prepend не решает проблему, потому что:

- Beeline выполняет маршрутизацию внутри своей AS
- выбор пути происходит до глобальной BGP-таблицы

---

## 💡 Решение

Скрипт реализует автоматическое управление маршрутами на FortiGate.

---

## 🧠 Архитектура решения

Whois (RIPE / RADB)
→ ASN организации
→ Префиксы (IPv4)
→ Суммаризация сетей
→ FortiGate API
→ Static routes / policy routes

---

## ⚙️ Логика работы

### 1. Получение ASN

- RIPE / RADB lookup

### 2. Получение префиксов

- сбор IPv4 префиксов ASN
- суммаризация сетей

### 3. Маршруты FortiGate

- next-hop → Beeline BGP peer
- interface → port3

### 4. Контроль BGP

- FortiGate API / SNMP

### 5. Динамика

| BGP | Action |
|-----|--------|
| Up  | routes enabled |
| Down | routes disabled |

---

## ⚙️ Конфигурация

APIBaseURL = https://<fortigate>/api/v2
APIToken = <token>
RIPE_ORG = ORG-ES15-RIPE
BGPNeighbor = 213.221.2.209
BGPNeighborPort = port3

---

## 🚀 Запуск

python main.py

---

## ⚠️ Ограничения

- нет IPv6
- нет кеша
- зависимость от RIPE/RADB
- лимиты FortiGate routes

---

## 🔐 Безопасность

- HTTPS API
- SNMPv3 рекомендуется
- ограничение доступа по IP
